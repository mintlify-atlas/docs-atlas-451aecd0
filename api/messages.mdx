---
title: Messages
description: Send and retrieve messages in chat rooms
---

Messages are stored in Redis with the room's TTL and broadcast in realtime to all connected users.

## Send message

<Note>
Requires authentication via `roomId` query parameter and `x-auth-token` cookie.
</Note>

Sends a message to a chat room. The message is stored in the room's history and broadcast to all connected users via the realtime channel.

<CodeGroup>
```bash cURL
curl -X POST "https://your-domain.com/api/messages?roomId=V1StGXR8_Z5jdHi6B-myT" \
  -H "Cookie: x-auth-token=your-token-here" \
  -H "Content-Type: application/json" \
  -d '{
    "sender": "Alice",
    "text": "Hello, everyone!"
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://your-domain.com/api/messages?roomId=V1StGXR8_Z5jdHi6B-myT',
  {
    method: 'POST',
    credentials: 'include',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      sender: 'Alice',
      text: 'Hello, everyone!'
    })
  }
);
```
</CodeGroup>

### Query parameters

<ParamField query="roomId" type="string" required>
  The unique identifier of the room where the message will be sent
</ParamField>

### Body parameters

<ParamField body="sender" type="string" required>
  The name of the message sender. Maximum length: 100 characters.
</ParamField>

<ParamField body="text" type="string" required>
  The message content. Maximum length: 1000 characters.
</ParamField>

### Response

Returns an empty response with status `200` on success.

### Validation rules

- `sender` must be a string with maximum 100 characters
- `text` must be a string with maximum 1000 characters
- The room must exist (returns error if room does not exist)

### Error responses

<Expandable title="Room does not exist">
```json
{
  "error": "Room does not exist"
}
```
</Expandable>

<Expandable title="Validation error">
```json
{
  "error": "Validation error"
}
```
</Expandable>

### Message structure

When a message is successfully sent, it's stored with the following structure:

<ResponseField name="id" type="string">
  Unique message identifier (generated using nanoid)
</ResponseField>

<ResponseField name="sender" type="string">
  The sender name from the request
</ResponseField>

<ResponseField name="text" type="string">
  The message content from the request
</ResponseField>

<ResponseField name="timestamp" type="number">
  Unix timestamp in milliseconds when the message was created
</ResponseField>

<ResponseField name="roomId" type="string">
  The room identifier
</ResponseField>

<ResponseField name="token" type="string" optional>
  The sender's auth token (only visible to the sender)
</ResponseField>

## Get messages

<Note>
Requires authentication via `roomId` query parameter and `x-auth-token` cookie.
</Note>

Retrieves all messages from a chat room's history.

<CodeGroup>
```bash cURL
curl -X GET "https://your-domain.com/api/messages?roomId=V1StGXR8_Z5jdHi6B-myT" \
  -H "Cookie: x-auth-token=your-token-here"
```

```javascript JavaScript
const response = await fetch(
  'https://your-domain.com/api/messages?roomId=V1StGXR8_Z5jdHi6B-myT',
  {
    credentials: 'include'
  }
);

const data = await response.json();
console.log(data.messages);
```
</CodeGroup>

### Query parameters

<ParamField query="roomId" type="string" required>
  The unique identifier of the room to retrieve messages from
</ParamField>

### Response

<ResponseField name="messages" type="array" required>
  An array of message objects in chronological order
  
  <Expandable title="Message object properties">
    <ResponseField name="id" type="string">
      Unique message identifier
    </ResponseField>
    
    <ResponseField name="sender" type="string">
      The name of the message sender
    </ResponseField>
    
    <ResponseField name="text" type="string">
      The message content
    </ResponseField>
    
    <ResponseField name="timestamp" type="number">
      Unix timestamp in milliseconds
    </ResponseField>
    
    <ResponseField name="roomId" type="string">
      The room identifier
    </ResponseField>
    
    <ResponseField name="token" type="string" optional>
      The auth token. Only present for messages sent by the requesting user (when the message token matches the request token).
    </ResponseField>
  </Expandable>
</ResponseField>

<Expandable title="Example response">
```json
{
  "messages": [
    {
      "id": "V1StGXR8_Z5jdHi6B-myT",
      "sender": "Alice",
      "text": "Hello, everyone!",
      "timestamp": 1709481600000,
      "roomId": "room-123",
      "token": "user-token-abc"
    },
    {
      "id": "a2bGz9Kp_Q8fgLx3M-pqR",
      "sender": "Bob",
      "text": "Hi Alice!",
      "timestamp": 1709481605000,
      "roomId": "room-123"
    }
  ]
}
```
</Expandable>

### Privacy note

<Info>
The `token` field is only included in messages that were sent by the requesting user. This allows the client to identify which messages belong to the current user without exposing other users' tokens.
</Info>
